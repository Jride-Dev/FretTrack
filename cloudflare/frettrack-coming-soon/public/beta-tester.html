FretTrack beta testing checklist

Use this package to test the repair-shop workflow and report useful, reproducible feedback. The checklist works in Excel, Google Sheets, LibreOffice, or any spreadsheet tool that can open CSV files.

Download CSV Checklist Open Beta Login

Who should use it

Shop owners, admins, techs, viewers, and invited testers who are checking real FretTrack workflows on desktop, mobile, tablet, and older browser devices.

How to test

Work through the rows with realistic but non-private shop data. Record the role, browser/device, result, severity, notes, and screenshots when something breaks.

How to submit

Use the in-app Report Issue control when you have app access, or send completed checklists, screenshots, and recordings to support@frettrack-app.com.

Status and severity values

Status

  • Fail

Severity

  • Low

  • High

Include with bugs

  • Browser and device
  • Shop role
  • Job, customer, part, vendor, or PO reference
  • Screenshot or video link
  • Whether it reproduces

CSV columns

Column How to fill it out
SectionThe checklist category, such as New Job Intake or Inventory Parts.
Test IDStable test ID like JOB-001 or PO-006.
Test ItemThe workflow to test.
Expected ResultWhat should happen if the workflow is working.
RoleSuggested role to test with.
StatusPass, Fail, Blocked, Not Tested, or Needs Clarification.
SeverityLow, Medium, High, or Critical.
Browser / DeviceExample: Chrome / Windows 11, Safari / iPadOS 17, Safari / iOS 12.
NotesWhat happened, what you expected, and any reproduction steps.
Screenshot / Video LinkAttach or link screenshots and short recordings for visual issues.
Reproducible?Yes, No, Sometimes, or Unknown.
TesterYour name or tester label.
DateDate tested.
