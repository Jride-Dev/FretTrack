# FretTrack Beta Tester Checklist

Thank you for testing FretTrack. Please use it like a normal repair intake tool, but keep this first pass small: one shop profile, one real-ish repair job, and a few photos.

## Test Steps

1. Can you log in?
2. Can you complete the shop onboarding profile?
3. Can you create or confirm your shop name, contact info, logo, state, and tax defaults?
4. Can you create a customer?
5. Can you create one repair job?
6. Can you upload 2-3 photos to the job?
7. Can you open/download the uploaded photos afterward?
8. Can you add parts?
9. Can you add services or labor?
10. Can you print or preview the repair/job sheet?
11. Can you find the customer/job again with customer lookup?
12. If you are an owner/admin, can you open Shop Settings and review shop members?
13. If testing a multi-user shop, can you add an existing signed-up user by email and set their role?
14. Can you create a 7-string guitar job and confirm setup/string fields make sense?
15. Can you create a 5-string bass job and confirm setup/string fields make sense?
16. Can you print the job sheet and confirm the instrument/string count appears correctly?
17. Can you sign out and sign back in?

## Feedback Questions

Please answer these after testing:

1. What confused you?
2. What felt slow or annoying?
3. Were any fields missing for a real repair intake?
4. Did anything feel unsafe, exposed, or too easy to change by accident?
5. Did the print sheet look usable for a customer or internal workbench?
6. What would stop you from using this in a real shop?
7. What is the one thing you would fix first?

## Report Format

For each issue, please include:

- What you were trying to do
- What happened
- What you expected
- The job number or customer name, if relevant
- A screenshot, if it helps
- Browser and device

## Feedback Labels

Feedback will be tracked in GitHub Issues with one or more of these labels:

- `beta-bug`: something broke or behaved incorrectly
- `beta-ux`: confusing, awkward, slow, or annoying workflow
- `beta-security`: access, privacy, cross-shop, or permission concern
- `beta-print`: printed/previewed sheets are wrong or unusable
- `beta-data`: missing, incorrect, duplicate, or hard-to-find data
- `post-beta`: useful, but not blocking this beta

## Fix Policy During This Test

Fix now:

- Login broken
- Onboarding broken
- Job save broken
- Photo upload broken
- Cross-shop/security issue
- Print sheet unusable

Defer:

- Prettier dashboard
- Accounting polish
- Inventory expansion
- SMS/email automation polish
- More reports
